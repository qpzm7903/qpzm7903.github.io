# 每日资讯博客

Hugo 静态博客，由本机的 Hermes agent 定时生成按话题拆分的每日资讯文章，推送到 GitHub 后自动构建发布，供作者每天阅读。

## Language

**话题（Topic）**:
一个可插拔的资讯领域配置单元（如 AI、科技、股票），拥有自己的信息源、发布节奏和模板。新增兴趣领域即新增一个话题。
_Avoid_: 领域、分类、栏目

**话题日报（Topic Daily）**:
某个话题在某一天产出的一篇独立博客文章。一个话题每天最多一篇；当天无内容则不发文，不存在空文章。
_Avoid_: 大合集、综合日报

**信息源（Source）**:
某个话题抓取内容的渠道（官方博客、官方仓库、HN、HuggingFace Daily Papers、财经 RSS 等），隶属于话题配置。
_Avoid_: 订阅、feed

**一手信息（First-hand）**:
由信息主体自己发布的内容（官方博客、官方仓库、论文原文、交易所数据）。话题的信息源准入原则；二手编译媒体不收。
_Avoid_: 原创内容

**精读条目（Featured Item）**:
话题日报开头深度导读的条目（每话题 1~3 条），含背景、核心内容与"为什么重要"，5~8 句。
_Avoid_: 头条、置顶

**快讯（Brief）**:
话题日报中 2~3 句实质摘要的普通条目。抓不到原文的条目直接丢弃，禁止"暂无摘要"。
_Avoid_: 简讯、速览

**主线（Storyline）**:
跨越多天日报条目的持续性故事（如"模型价格战"），周报的基本单元；名字一经确立跨周沿用。
_Avoid_: 话题（已有他义）、专题

**概念卡（Concept Card）**:
`content/concepts/` 下一篇独立于时效的术语解释文章（人话版 + 例子），由日报解释概念时沉淀，日报正文链接它。
_Avoid_: 词条、百科
