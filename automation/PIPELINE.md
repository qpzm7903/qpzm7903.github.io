# 每日日报流水线规章

本文件是流水线执行者（Hermes agent）每次运行必须遵守的编辑部规章。话题清单和各话题的信息源见 `automation/topics/*.yaml`。修改本文件或话题配置即可调整流水线行为，无需改动 Hermes 侧技能。

## 执行流程

每次被调度或手动触发时：

1. `git pull --rebase` 确保仓库最新。
2. 读取 `automation/topics/` 下所有 YAML，得到话题清单。
3. 对每个话题判断今天是否应发：
   - `cadence: daily` → 每天发。
   - `cadence: trading-days` → 仅当 A股或美股至少一个市场今天（或美股昨夜）有交易时发。
   - **幂等**：若 `content/posts/daily-<slug>-YYYY-MM-DD.md` 已存在，跳过该话题（补跑安全）。
4. 逐话题：抓取信息源 → 筛选 → 撰写 → 落盘。
5. 全部话题完成后，运行 `hugo --minify` 验证构建（exit 0 且无 ERROR），失败则修复或回退该篇，**禁止把坏文章推上去**。
6. `git add content/posts/ && git commit && git push`。push 被拒则 `git pull --rebase` 后重试一次。
7. 输出微信简讯（见「通知格式」）。

## 内容红线（逐条硬性）

- **一手信息原则**：只收信息主体自己发布的内容。二手编译媒体一律不收。
- **实抓原文**：每个条目必须实际抓取原文页面后再写摘要。抓不到原文的条目直接丢弃。**禁止出现"暂无摘要"或模板化的空话**（如"官方源更新，说明生态有实质动作"）。
- **全中文输出**：正文全部中文；条目保留英文原标题 + 链接。
- **宁缺毋滥**：配置里的条数是上限不是配额。某话题当天筛完不足 3 条合格条目且无精读价值时，当天不发该话题。
- **跨天去重**：写作前检查本话题最近 7 天已发文章中出现过的链接，已报道过的条目跳过（同一事件有实质新进展除外，需注明"进展"）。
- **跨话题去重**：同一条目命中多个话题时归入 AI 优先，科技话题不重复收录。

## 文章格式

文件名：`content/posts/daily-<slug>-YYYY-MM-DD.md`

Front matter（强制，缺失即事故）：

```yaml
---
title: "<emoji> <话题名>日报 | YYYY-MM-DD"
date: YYYY-MM-DDTHH:MM:SS+08:00   # 必须是实际生成时刻（用 date 命令取当前北京时间）。
                                  # 绝不允许写未来时间：本站未开 buildFuture，
                                  # 未来日期的文章不会被构建（= 隐形文章事故）。
draft: false
tags: [<话题 YAML 中的 tags>]
categories: ["<话题 YAML 中的 category>"]
---
```

正文结构由各话题 YAML 的 `structure` 字段定义。通用约定：

- 「⭐ 今日最值得关注」= 精读条目区：每条 5~8 句的深度导读（背景、核心内容、为什么对你重要）。
- 快讯条目：2~3 句实质摘要，说清"是什么 + 有什么新东西"，附原文链接。
- 标题不堆砌 emoji，每个二级标题最多一个。

## 发布

- commit message：`发布<话题名>日报 YYYY-MM-DD`，多话题一次 commit 时用 `发布日报 YYYY-MM-DD（AI/论文/科技/股票）`。
- 直接推 `master`，GitHub Actions 自动构建部署。
- 文章 URL：`https://qpzm7903.github.io/posts/daily-<slug>-YYYY-MM-DD/`

## 通知格式

流水线结束时的最终输出会被投递到微信，格式：

```
📮 日报 YYYY-MM-DD
✅ AI：<文章标题级一句话> <URL>
✅ 论文：… <URL>
✅ 科技：… <URL>
➖ 股票：休市未发
```

任何环节失败（源全挂、构建失败、push 失败等）必须在简讯开头用 `⚠️` 明确报告失败话题和原因，**不允许静默失败**。

## 手动触发

在任何与 Hermes 的对话（含微信）里说"补跑今天的日报"或"跑一下 AI 日报"即可；幂等规则保证不会重复发文。
