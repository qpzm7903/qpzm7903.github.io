# 用 Hermes 按话题独立成文重建每日资讯流水线

旧 OpenClaw 流水线（跑在另一台 Mac）从 2026-03-09 起持续生成缺失 front matter 的日报，站点上不可见却无人知晓，暴露出单体大日报 + 无反馈回路的脆弱性。决定改用本机 Hermes agent 每天 07:00 生成日报：每个话题（AI、AI 论文、科技、股票）独立成文而非合成一篇大日报，话题配置（信息源、条数上限、模板）作为 YAML 存放在本仓库 `automation/topics/` 而非 Hermes 侧，生成后本地跑 `hugo --minify` 校验构建再直接 commit/push master 自动发布，并通过微信发送成功简讯与失败告警。

## Considered Options

- **每天一篇大合集**（旧模式）：被否——话题间互相耦合，单一源故障拖垮整篇，空章节难看，且股票与科技的发布节奏天然不同。
- **配置放 Hermes 侧**：被否——失去版本历史，换机器迁移成本高；配置入仓后 Hermes 侧只剩"拉仓库、按配置执行"的薄技能，未来迁去 VPS 几乎零成本。
- **中文科技媒体作为信息源**：被否——坚持一手信息原则（官方博客/仓库、HN、HF Daily Papers、交易所数据），中文阅读体验由流水线自己翻译总结保证。

## Consequences

- 工具版本追踪（Claude Code / Codex / OpenClaw）不再独立发文，并入 AI 日报的「工具版本」小节；旧的 163 篇版本帖停止增长。
- 发文前的 hugo 构建校验 + 微信失败告警是防止"隐形坏文章"事故重演的两道保险，任何后续改动不得移除。
- 旧 Mac 上的 OpenClaw 定时任务必须手动关停，否则会与新流水线双重发布。
