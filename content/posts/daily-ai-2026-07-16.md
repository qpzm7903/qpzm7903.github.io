### 1. Thinking Machines 发布首个开源权重模型 Inkling：975B 参数 MoE、可听可看、可微调

Mira Murati 创立的 Thinking Machines 发布了公司的第一个模型——Inkling，并以完整开放权重的形式发布。

关键参数：

- **架构**：[MoE](/concepts/moe/) Transformer，总参 975B，激活参 41B。MoE 设计借鉴 DeepSeek-V3：每层 256 个路由专家 + 2 个共享专家，每个 token 激活 6 个路由专家。路由用 sigmoid + 无辅助损失的负载均衡 bias。
- **上下文**：1M tokens。
- **预训练**：45 万亿 token 的文本、图像、音频和视频。
- **多模态原生**：不用外部编码器——音频输入为 dMel 频谱图，图像编码为 40×40 像素 patch 经四层 hMLP。模型直接做语音转写、语音指令跟随、录音问答、视觉推理。
- **可控思考力度**：effort 参数可从 0.2 调到 0.99——在 Terminal Bench 2.1 上用约 1/3 的 token 就能匹配 Nemotron 3 Ultra 的得分，成本和延迟显著低于同类。
- **同发 Inkling-Small**：12B 激活，低延迟更便宜。

论文同时给出了超长 benchmark 表，横跨 agentic coding、推理、指令跟随、事实性、视觉、音频六个维度。没有追求单一 benchmark 上的第一，主打"广、平衡"——Inkling 不是最强模型，但可能是目前最适合做微调底座的开源模型：多模态、可控思考、可在 Tinker 上微调。完整权重已在 HuggingFace (`thinkingmachines/inkling`) 发布，含原装 checkpoint 和 NVFP4 checkpoint（适用于 Blackwell 推理）。发布方甚至让 Inkling 自己在 Tinker 上对自己做微调，来演示自定义能力。

> 原文金句：「Inkling is not the strongest overall model available today, open or closed. Instead, a combination of qualities makes it a good open-weights base for customization: multimodal capabilities, efficient thinking, and availability on Tinker for fine-tuning.」

"Inkling 不是当今最强的模型——无论开源还是闭源。但几种品质的组合让它成为最好的开源微调底座：多模态能力、高效可控的思考、以及 Tinker 上的微调可用性。"

### 2. SpaceXAI 把 Grok Build 开源了：终端内 AI coding agent，对标 Claude Code/Codex

SpaceXAI 把 Grok Build 的 Rust 源码发布到 GitHub（`xai-org/grok-build`），仅供 macOS / Linux / Windows 安装。这是 xAI 旗下跟进 Claude Code、Codex、OpenCode 之后第四个开源的"终端 AI coding agent"。

核心信息：

- **定位**：full-screen TUI 的 AI coding agent，读代码库、编辑文件、执行 shell 命令、联网搜索，支持长时间运行任务。三种使用模式：交互式（TUI 全屏）、headless（脚本 / CI）、或通过 Agent Client Protocol（ACP）嵌入编辑器。
- **License**：仓库带 LICENSE 文件（具体许可证待确认），安装方式 `curl -fsSL https://x.ai/cli/install.sh | bash`。
- **Codebase**：Rust 写的 CLI/TUI + agent runtime，synced periodically from SpaceXAI monorepo。

context 包含两个时间点：昨天 Grok Build 被安全研究员 Cereblab 曝光上传整个代码库到 Google Cloud（7/15 AI 日报已报道，Musk 承诺删除），今天 xAI 反手把 harness 本身开源。两条拼在一起意味着 xAI 决定正面参与 AI coding 军备赛，而不是在产品边界上含糊。

- 仓库：[github.com/xai-org/grok-build](https://github.com/xai-org/grok-build)
- HN 讨论（139 分）：[https://news.ycombinator.com/item?id=48926590](https://news.ycombinator.com/item?id=48926590)

### 3. OpenAI 首款品牌硬件不是那个智能音箱，是一台 $230 的发光键盘

正当人们还在消化 OpenAI 会做智能音箱的传闻时，OpenAI 今天上线了第一款真正开卖的品牌硬件——Codex Micro，一台 $230 的 RGB 迷你键盘。

是什么：

- 与 Work Louder 合作的"限量版"——键盘上排 6 个磨砂按键循环变色，监控 6 路 Codex agentic thread 的状态：白=空闲、蓝=思考中、绿=完成、琥珀=需要反馈、红=出错。一键点亮的按键直接把对应 Codex 窗口拉到前台。
- 下排 6 键默认映射常见 Codex 操作——接受/拒绝修改、分支 thread、一键 talk。可通过软件重映射，并附 32 个可替换键帽。
- 可切换 5 层函数集，其他层留给通用桌面快捷键。
- Work Louder 本来就有几乎同款的 Creator Micro 键盘，Codex Micro 主要是 6 个 MQTT 式状态灯 + OpenAI 品牌贴纸。

这意味着什么：OpenAI 开始押"多 agentic thread 桌面前台"的工作流——先前有人发现 dev 在咖啡馆半开笔记本盯 agent 任务进度，正是 Codex Micro 想取代的场景。稳态级 coders 同时跑几个 Codex agent 反复迭代，键盘状态灯让你不用切回屏幕也能一眼感知哪个 thread 出错或需要反馈。和 Jony Ive 那个去屏幕化的智能音箱是两条产品线——这是"先卖出一个贴牌键盘" 的实用进路。

- 原文：[OpenAI's first branded hardware is... a light-up keyboard?](https://arstechnica.com/ai/2026/07/openais-first-branded-hardware-is-a-light-up-keyboard/)

## 📖 今日英语

**open weights** — "Today we are advancing our mission by releasing a model we trained from scratch with the full **weights** available, so that people can make it their own."（出处：Thinking Machines Inkling 发布文章）

"开放权重"——公开发布模型的全部参数（权重张量），让任何人下载、自部署、再微调。和"开源"细微区别：许多公司会用"open weights"而非"open source"——因为训练数据、训练代码未必一起发布。常见搭配：release open weights、open-weights model、weights on Hugging Face。

**controllable thinking effort** — "**controllable thinking effort**, allowing you to balance performance with token efficiency... Inkling spends one third as many tokens to achieve the same performance as Nemotron 3 Ultra on Terminal Bench."（出处：同上）

"可控思考力度"——模型推理时可以调节"思考深度"的超参，低 effort 省成本和高 effort 冲满能力之间自由切换。类似 GPT 的 reasoning effort 设定。好模型的衡量标准从此不止是"峰值分"——还包括在不同 effort 下的性价比曲线。常见搭配：scale effort、low-effort inference、effort parameter。

**harness** — "SpaceXAI's terminal-based AI coding **agent harness**... and its agent runtime."（出处：Grok Build 仓库 README）

"agent harness"——让 AI agent 跑起来的运行壳子。类比马具（harness）套住马匹使之拉动车辕——harness 套住底层模型，提供工具接口、上下文管理、用户交互等。在 Claude Code、Codex、OpenCode、Grok Build 这些 coding agent 里，harness 不是模型本身，而是模型与文件系统、终端、web 搜索之间的胶水层。常见搭配：agent harness、harness prover、coding harness。

**noncompliance / compliance** — "Cognition evaluated the model on their Propaganda and Censorship Eval... it exhibited strong patterns of censorship **noncompliance**."（出处：Inkling 文章中提及第三方安全评估）

"不合规 / 不合规性"——在此语境指模型拒绝回答本应拒绝的请求（如配合审查），或反之去应答本应拒绝的内容。一个不合规的模型指它在本该合规的场景下被评测为不合规。这是 open-weight 模型发布方披露的重要安全维度之一。常见搭配：censorship noncompliance、regulatory compliance、compliance rate。

**perpetual license** — "Broadcom's changes, which include eliminating **perpetual licenses** in favor of subscriptions to large bundles, forced the retail chain's hand."（出处：Ars Technica Sheetz/VMware 报道，同日科技日报）

"永久许可证"——一次付费后可永久使用的软件许可模式，与"订阅"（subscription）相对。Broadcom 收购 VMware 后砍掉永久许可、改推订阅捆绑包，引发大批企业客户出走。open-weight 模型走的是另一种"永久"——权重本身永久可用，不靠 API 订阅。常见搭配：perpetual license、perpetual vs subscription、buy perpetual。
