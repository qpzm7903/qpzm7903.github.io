### Qwen 3.8 发布：2.4T 参数，开放权重即将放出

阿里巴巴 Qwen 团队昨日（7/19）官方 Twitter 宣布 [Qwen3.8](https://twitter.com/Alibaba_Qwen/status/2078759124914098291) 启动发布。这是继上月 Kimi K3（2.8T 参数）后又一个走进万亿参数俱乐部的中国大模型。

几个关键信息：

- **2.4T 参数**（约 2.4 万亿），官方自称"仅次于 Anthropic Fable 5"，定位在 frontier 层级。
- **开放权重即将放出**（"going open-weight soon"），延续 Qwen 系列一贯的开源承诺，但具体日期未给。
- **现在就能体验的版本是 Qwen3.8-Max-Preview**，已在阿里云 Token Plan、Qoder 和 QoderWork 上开放试用；国内入口走 `platform.qianwenai.com`，国际入口走 `qwencloud.com`。
- "continuously evolving"（持续进化）——暗示后面几天/几周会持续放出正式版与开放权重，首发只是 preview。

为什么对你重要：Qwen3.8 K3 是 2026 年 7 月中国 AI 实验室连续推向万亿参数级别的第二个，把"open-weight + frontier 能力 + 中文实验室"这条赛道做成了一种可观察的趋势。对开发者实际意味着两件事：一是**选模型时除了 Kimi K3 / GLM-5.2 又多一个候选**，尤其是需要中文场景 + 国际合规弹性时，Qwen 长期合作社区底子最厚（生态最规范）；二是**开放权重版本放出后 Hugging Face / ModelScope 上会立刻有人开始做本地部署 + 后训练**，如果你近期有私有化部署计划，留挨 open-weight 的官方声明比较划算。

> 原文金句："Continuously evolving... second only to Fable 5"
> 对照："持续演进……仅次于 Fable 5"——一句话同时划清了能力定位（frontier）和差距方向（仍在追赶 Anthropic）

### Claude Code 的运行底层换成 Bun 的 Rust 端——Simon Willison 实证

Simon Willison 昨日发布 [短文](https://simonwillison.net/2026/Jul/19/claude-code-in-bun-in-rust/) 引爆 HN（[357 分](https://news.ycombinator.com/item?id=48960206)）。

背景和动作：

- Bun 作者 Jarred Sumner 在官方博客 [Rewriting Bun in Rust](https://bun.com/blog/bun-in-rust) 里称："Claude Code v2.1.181（6 月 17 日）及之后版本使用 Bun 的 Rust 端口，Linux 启动快了 10%，其他没什么人注意——Boring is good（太平无事是好事）。"
- Simon Willison 决定亲自验证——在自己的 Claude Code 安装里跑 `strings ~/.local/bin/claude | grep -m1 'Bun v1'`，输出 `Bun v1.4.0 (macOS arm64)`——这是比 Bun 在 GitHub 已发版本（v1.3.14，5 月 12 日）高一档的预览版，对应 Bun 已经放出的 canary 通道。
- 再跑 `strings ~/.local/bin/claude | grep -Eo 'src/[[:alnum:]_./-]+\\.rs'` 拿出 563 个 Rust 源文件路径——直接证明 Claude Code 内嵌的 Bun runtime 已是 Rust 端口。
- 结论是：Bun（JS runtime）的 Rust 端口已经在"数百万设备"上跑 Claude Code，没有被开发者察觉——"Boring is good"成立。

为什么对你重要：这是大规模 JS runtime 编译到 Rust 并悄悄替换在途分发的最近一个案例。对大多数开发者**直接含义**是：未来安装的 Claude Code 在启动和内存上的小步提升并不必留到升级汇报里去找——靠 Bun 的 Rust 端口已经默默吃掉部分。更深一层的含义：当一款边缘 JS runtime 在 CLI 工具里内嵌分发而不是让你装 Node/Bun，工具方实际上"把 runtime 和 product 一次性包起来发"——这也是 2025 年以来 agent CLI 工具普遍多做的一件事（Claude Code、Codex CLI、oh-my-pi 都走了 bundled runtime，路径稍有不同）。

> 原文金句："Startup got 10% faster on Linux but otherwise, barely anyone noticed. Boring is good."
> 对照："Linux 启动快了 10%，其他几乎没人察觉——太平无事，才是好事。"

### Moonshot 因 Kimi K3 需求过大暂停新订阅；SemiAnalysis 指出它的硬件需求反而更大

月之暗面 Kimi K3 在 7/17 发布后热度异常。本周来自三条线的信息：

- **新订阅暂停**：[HN 报道](https://news.ycombinator.com/item?id=48958150) 引用 Moonshot 的官方 Twitter（[@kimi_moonshot](https://twitter.com/kimi_moonshot/status/2078855608565207130)），宣布因需求过大暂停新订阅，先解决存量用户的稳定性。算法层面 Moonshot 主打的 [KDA（Key-Driven Attention）](https://www.kimi.com/blog/kimi-k3) 是线性注意力变体，被认为可能"降低对 GPU 的依赖"。
- **SemiAnalysis 反向研究**：[华尔街见闻引述 SemiAnalysis 报告](https://wallstreetcn.com/articles/3777348) 给了反直觉判断——K3 chosen 的 KDA 机制**提高**了注意力效率，但因为**参数量巨大（2.8T）+ 推理架构需求**，对 GPU、HBM、DRAM 和网络的需求反而**更多**，而非更少。这帮某短线叙事泼了冷水——"线性注意力=硬件减负"的直觉不成立，K3 是用"内存里存更多来换计算量省"，一样占硬件。
- **高盛点评**：[Kimi K3 达到全球普及关键点](https://wallstreetcn.com/articles/3777291)，但高端编程赛道竞争激烈；定价上被抬到 Claude Sonnet 档（$3/$15）是中国 AI 实验室迄今为止最贵的一款发布。
- **市场反应**："Mag7 资本支出若遭削减，更广泛回调或难以避免"——[美银 Hartnett](https://wallstreetcn.com/articles/3777361) 把 K3 这种价位选型当作"AI 高估值的近期警示旗"之一。

为什么对你重要：这是 2025 年下半年以来国内 AI 实验室"open-weight 大模型冲万亿参数 + 向企业定价"主线最不像玩具的一次具体进展。对你选型与买 GPU/上云有三个具体含义：① K3 这类模型**便宜不到哪去**（开放权重但运行硬件更贵，自托管未必比 API 省钱）。② API 定价虽然压在 Anthropic Sonnet 档，但百万实际会提货的人流多到暂停订阅，说明"定价还是有现实压力"，不一定有富余下调空间。③ 不要被"线性注意力" 这个工程名词带跑——量化到 GPU/HBM/网络才知道是减是加。

> 原文金句："K3's massive parameter scale and inference architecture demands **may require more** high-end AI hardware, not less."
> 对照："K3 庞大的参数规模和推理架构产物，对高端 AI 硬件的要求**可能反而更大**，而不是更小。"

## 📖 今日英语

- **"Boring is good"**（太平无事才是好事）——Simon Willison 引用 Jarred Sumner 在 Bun Rust 端口上线后的话。在软件发布语境，"boring" 的褒义是"不动声色地替换底层"，背后含义是"用户完全没察觉就完成了迁移"。
- **"continuously evolving"**（持续演进）—— Qwen 官宣语。强调发布是个延续过程，不是一次性 drop——这在"open-weight soon"这种不确定时间承诺里很常用。
- **"frontier AI models"**（前沿 AI 模型）——Qwen 用 "second only to Fable 5" 自我定位时用的术语，指当下能力最强那几个模型；引用例句："second only to Fable 5"
- **"Intelligence per dollar"**（每美元智能）—— 英伟达针对 Vera Rubin 换叙述时用的核心指标，从传统"每美元 token 数"转向"每美元可产生的智能产出"，成为新阶段企业 ROI 的中央衡量项。
- **"policy-adaptive image guardrails"**（策略自适应图像护栏）—— 见论文日报 PolicyShiftGuard；指护栏不是固定的"Unsafe / Safe"分类，而是按"当下有哪些条策略"动态判定是否违规。
